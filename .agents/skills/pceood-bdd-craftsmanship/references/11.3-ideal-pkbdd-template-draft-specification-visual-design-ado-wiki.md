
# This is a page template for drafting a new Visual Design Specification in Azure DevOps Wiki

**IMPORTANT NOTES:**
1. Copy the contents of this template into a new Visual Design Specification page.
2. Fill in the necessary details as instructed.
3. Only when the specification is being promoted to `Approved New` should empty and unneeded sections be deleted.
4. This page is only for visual design concerns such as layout, composition, styling, spacing, and visual arrangement.
5. This page must not be used to define logic, state transitions, triggers, or behavioral rules. Those belong in behavioral specifications.
6. If there is any conflict between this page and a behavioral specification, the behavioral specification governs.

# IMPORTANT RULES
1. Visual design is only authoritative for things that are NOT specified in the behavioral specifications.
2. If there is any conflict, the behavioral specification is the source of truth.

---

# General guidance for Separation of Concerns between behavior and visual design
> - If a section reads like logic (“when the viewport width is below X, move the nav to…”) then it belongs in a behavior specification, even if you reference specific design frames.
> - If it reads like composition / styling (“header at top, nav down the left, content centered with max width Y”) then it belongs in this visual design specification.
