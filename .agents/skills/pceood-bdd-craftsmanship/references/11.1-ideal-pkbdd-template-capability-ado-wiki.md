
# This is a Capability page template for Azure DevOps Wiki

**IMPORTANT NOTES:**
1. Copy the contents of the markdown block below and paste it into your new Capability page.
2. Provide a summary description of the capability in the `# Description` section.
3. For each unique stakeholder this capability enables, add an entry in the `# What This Capability Enables` section.
4. Preferred production shape: when the capability has child pages, provide curated child-page links explicitly.
5. Human authoring allowance: when manual maintenance burden would make curated child-page listings unsafe or inefficient, `[[_TOSP_]]` may be used as the navigation surface temporarily or operationally.
6. AI authoring expectation: when an AI Capability Manager is producing or revising the page, it should prefer curated child-page listings rather than relying on `[[_TOSP_]]`.

Use one of the following navigation approaches:
- preferred: curated child-page links
- allowed for human-maintained pages where manual upkeep is too costly: `[[_TOSP_]]`
- do not leave both as active final navigation surfaces unless there is a deliberate reason to do so
