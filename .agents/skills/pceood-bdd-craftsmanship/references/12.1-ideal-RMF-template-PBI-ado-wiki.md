
# This is a template for drafting a new Standard Product Backlog Item (PBI) in Azure DevOps Wiki

**IMPORTANT NOTES:**
1. ADO board item _templates_ use single-line field values, even if the field is a multi-line text field. So, in order for it to add multi-line text to the board items, each line after the first line **_must_** start with `<br />`.
2. Following are the `Definition of Ready` and `Definition of Done` field values used in a `Standard PBI`.

## Definition of Ready
```markdown
#### **Product Exit Criteria**
<br />In order for Product to release responsibility for this work item to Engineering, the following criteria must be met:
<br />- [ ] All specifications listed and linked to the PKB
<br />- [ ] All specifications in the **Approved New**, **Committed**, **Past Attempted**, or **Current** state
<br />- [ ] All dependencies have been listed
<br />- [ ] Definition of done approved by Product Owner
<br />- [ ] The Product Owner believes everyone involved shares the same understanding of the Definition of Done
<br />
<br />---
<br />
<br />#### **Engineering Entrance Criteria**
<br />In order for Engineering to accept responsibility for this work item from Product, the following criteria must be met:
<br />- [ ] The Development Team believes everyone involved shares the same understanding of the Definition of Done
<br />- [ ] Definition of done accepted by the Development Team
<br />- [ ] All dependencies have been resolved
<br />&nbsp;&nbsp;- [ ] - &lt;list dependencies here; if none, mark the parent bullet with ` (None)` and delete this one&gt;

```

## Definition of Done
```markdown
#### **Specifications**
<br />In order for the PBI to be complete, the following scenarios have been deployed to and tested in the DEV environment:
<br />- [ ] [Specification]()
<br />
<br />---
<br />
<br />#### **Engineering Exit Criteria**
<br />In order for Engineering to release responsibility for this work item to Product, the following criteria must be met:
<br />- [ ] Team has verified that all the listed criteria have been met
<br />- [ ] Changes have been promoted to TEST
<br />
<br />---
<br />
<br />#### **Product Entrance Criteria**
<br />In order for Product to accept responsibility for this work item from Engineering, the following criteria must be met:
<br />- [ ] The product meets each scenario in each specification listed above
<br />- [ ] As the Product Owner, I attest that the above have been proven to me
```
