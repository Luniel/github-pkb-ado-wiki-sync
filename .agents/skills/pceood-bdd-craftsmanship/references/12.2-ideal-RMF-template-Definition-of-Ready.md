# Definition of Ready Template

## For work items to create or change system functionality

### Jira (standard)

```Markdown
# Definition of Ready

## Product Exit Criteria
**In order** for Product to release responsibility for this work item to Engineering, the following criteria **must be met**:
* [  ] All specifications are listed in the DoD and linked to the PKB
* [  ] All specifications are in the **Approved New**, **Committed**, **Past Attempted**, or **Current state**
* [  ] Defective behaviors discovered during maturation of the requirements for this work item have been identified to be escalated to Product Management for consideration separate from this work item
* [  ] Dependencies are defined (list dependencies as Engineering Entrance Criteria)
* [  ] The DoD is approved by Product Owner
* [  ] The Product Owner believes everyone involved shares the same understanding of the Definition of Done

## Engineering Entrance Criteria
**In order** for Engineering to accept responsibility for this work item from Product, the following criteria **must be met**:
* [  ] The Development Team believes everyone involved shares the same understanding of the Definition of Done
* [  ] The DoD is accepted by the Development Team
* [  ] All external dependencies have been resolved
   * [  ] <list all dependencies here>
* [  ] A sufficient test data set is available
* [  ] This work item has been estimated by the team
* [  ] This work item is small enough to be completed within a single Sprint

## Sprint Entrance Criteria
* [  ] The team has all the necessary skills available in the targeted sprint
* [  ] The team has enough capacity in the targeted sprint

## Post-Commitment Tasks
* [  ] POs: Promote specifications from Approved New to Committed when committed to a Sprint
```

## For work items to create automated tests for existing functionality

### Jira (standard)

```Markdown
# Definition of Ready

## Product Exit Criteria
**In order** for Product to release responsibility for this work item to Engineering, the following criteria **must be met**:
* [  ] All test cases to be automated are listed in the DoD and linked to the PKB along with draft PKB pages for the team to document any Gherkin specifications
* [  ] All test cases to be automated have been determined, by the team, as ready for automation
* [  ] Defective behaviors discovered during maturation of the requirements for this work item have been identified to be escalated to Product Management for consideration separate from this work item
* [  ] Dependencies are defined (list dependencies as Engineering Entrance Criteria)
* [  ] The DoD is approved by Product Owner
* [  ] The Product Owner believes everyone involved shares the same understanding of the Definition of Done

## Engineering Entrance Criteria
**In order** for Engineering to accept responsibility for this work item from Product, the following criteria **must be met**:
* [  ] The Development Team believes everyone involved shares the same understanding of the Definition of Done
* [  ] The Development Team has identified refactoring that might be needed
   * [  ] Expected refactoring has been communicated to any relevant external stakeholders
* [  ] The DoD is accepted by the Development Team
* [  ] All external dependencies have been resolved
   * [  ] <list all dependencies here>
* [  ] A sufficient test data set is available
* [  ] This work item has been estimated by the team
* [  ] This work item is small enough to be completed within a single Sprint

## Sprint Entrance Criteria
* [  ] The team has all the necessary skills available in the targeted sprint
* [  ] The team has enough capacity in the targeted sprint

## Post-Commitment Tasks
* [  ] POs: Promote specifications from Approved New to Committed when committed to a Sprint
```
