# Definition of Done Template

## For work items to create or change system functionality

### Jira (standard)

```Markdown
# Definition of Done

## Specifications

In order for the PBI to be complete, the following scenarios have been deployed to and tested in the DEV environment, according to the [implementation guide]():
* [  ] <add link(s) to specification(s) here>
   * [  ] <creating structure here can help>
   * [  ] <don't forget the square brackets to track completion>

## Engineering Exit Criteria

In order for Engineering to release responsibility for this work item to Product, the following criteria must be met:
* [  ] Automated unit tests written
* [  ] Any code changes have been reviewed by non-author
* [  ] Test cases for the requirements are passing in DEV
* [  ] Every specification in the PKB that is marked as Current (and should therefore not be changed as part of this PBI) has not been broken
* [  ] Team has verified that the below-listed criteria have been met (see the engineering acceptance guide)

## Product Entrance Criteria

In order for Product to accept responsibility for this work item back from Engineering, the following criteria must be met:
* [  ] The product meets each scenario in each specification listed above
* [  ] All required unit tests exist
* [  ] All required manual tests exist
* [  ] As the Product Owner, I attest that the above have been proven to me in accordance with the [Product Management Acceptance Guide]()
* [  ] Specifications in the PKB have been promoted to the correct state (_i.e._, **Past Attempted** or **Current**)
```

## For work items to create automated tests for existing functionality

### Jira (standard)

```Markdown
# Definition of Done

## Specifications

In order for the PBI to be complete, automated tests for the following scenarios have been created and passed in the target environment, according to the [Implementation Guide for Automated Tests]():

* [  ] <add link(s) to specification(s) here>
   * [  ] <creating structure here can help>
   * [  ] <don't forget the square brackets to track completion>

## Engineering Exit Criteria

In order for Engineering to release responsibility for this work item to Product, the following criteria must be met:

* [  ] All automated test code created or changed as a result of this work item has been peer-reviewed per the [Test Automation Standards]()
* [  ] All automated tests do pass
* [  ] The team has verified that all the criteria above have been met (see the [Engineering Acceptance Guide]())
* [  ] Any defective system behavior discovered during automation has been documented as part of this work item and communicated to the Product Owner
* [  ] Any corresponding Gherkin that was created as part of the work item:
   * [  ] Has been documented as part of the PKB
   * [  ] Has been reviewed with the PO

## Product Entrance Criteria

In order for Product to accept responsibility for this work item back from Engineering, the following criteria must be met:

* [  ] Each scenario in each specification listed above have automated tests that pass
* [  ] Any defective system behavior discovered during automation has been communicated to me
* [  ] As the Product Owner, I attest that the above have been proven to me in accordance with the [Product Management Acceptance Guide]()
* [  ] As the Product Owner, I understand any Gherkin specifications created by the team and take responsibility for its curation in the PKB
* [  ] Specifications in the PKB have been promoted to the correct state (_i.e._, **Past Attempted** or **Current**)
```
