# Canonical Method: Permutations Analysis

1. Identify all parameters in Given.
2. Identify all parameters in When.
3. Identify implied parameters in Then.
4. Generate values and variations for those parameters.
5. Identify meaningful permutations.
6. Process the discovered permutations.

## Parameter Variation Examples
- values
- presence or absence
- cardinality
- formatting
- combinations
- whether a line exists at all
- whether an actor or condition is present
- whether a setting, locale, date, environment, configuration, or dependency state changes outcome

## Required Disposition Set
- create now
- defer
- reject
- research
- current-spec correction
- upstream dependency candidate
- downstream realization concern
- additional aspect candidate
- support artifact / decision note
